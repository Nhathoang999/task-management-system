# BUG-002: Search ignores leading spaces

**Severity**: Minor
**Priority**: Medium

## Steps to Reproduce
1. Create a task with the title "Meeting Notes".
2. Type " Meeting" (with a leading space) in the search box.

## Expected Result
The search should trim leading and trailing spaces, or match the literal string. It should find "Meeting Notes" if trimming is applied.

## Actual Result
The search returns "No tasks found" because it does not trim the input and treats the leading space as a literal character that doesn't match the database index accurately.

## Root Cause
The frontend search input is directly passing `e.target.value` without applying `.trim()`, and the backend `LIKE` query fails to match.
