# BUG-001: Delete dialog closes unexpectedly

**Severity**: Major
**Priority**: High

## Steps to Reproduce
1. Login to the application.
2. Click the 'Delete' button on any existing task.
3. When the confirmation dialog appears, click anywhere outside the dialog (backdrop).

## Expected Result
The dialog should either require explicit cancellation (Cancel button) or if it closes, it should NOT delete the task.

## Actual Result
The dialog closes and the task is immediately deleted without user confirmation.

## Root Cause
The `onClose` event of the MUI Dialog is triggering the `handleDelete` method instead of `handleCancel`, or the backdrop click is bubbling up to the delete action.
